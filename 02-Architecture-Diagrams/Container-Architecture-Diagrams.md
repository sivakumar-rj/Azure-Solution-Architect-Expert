# Container & Kubernetes Architecture Diagrams

## 1. Docker Architecture

```
┌─────────────────────────────────────────────────────────────┐

                         ▼
┌─────────────────────────────────────────────────────────────┐
│                      Docker Daemon                           │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │   Images     │  │  Containers  │  │   Networks   │      │
│  │  (Templates) │  │  (Running)   │  │   (Bridge)   │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │

│  └──────────────┘  └──────────────┘  └──────────────┘      │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼

```

## 2. Container Lifecycle

```
┌──────────────┐
│ Dockerfile   │
└──────┬───────┘
       │ docker build
       ▼
┌──────────────┐
│    Image     │ ◄──── docker pull (from Registry)
└──────┬───────┘
       │ docker run
       ▼
┌──────────────┐
│  Container   │
│  (Created)   │
└──────┬───────┘
       │ docker start
       ▼
┌──────────────┐
│  Container   │ ◄──── docker restart
│  (Running)   │
└──────┬───────┘
       │ docker stop
       ▼
┌──────────────┐
│  Container   │
│  (Stopped)   │
└──────┬───────┘
       │ docker rm
       ▼
┌──────────────┐
│   Removed    │
└──────────────┘
```

## 3. Multi-Stage Docker Build

```
┌─────────────────────────────────────────────────────────────┐
│                    Stage 1: Builder                          │
│  ┌────────────────────────────────────────────────────┐     │
│  │  FROM node:18 AS builder                           │     │
│  │  WORKDIR /app                                      │     │
│  │  COPY package*.json ./                             │     │
│  │  RUN npm install                                   │     │
│  │  COPY . .                                          │     │
│  │  RUN npm run build                                 │     │
│  │                                                    │     │
│  │  Size: 1.2 GB (includes build tools)              │     │
│  └────────────────────────────────────────────────────┘     │
└────────────────────────┬────────────────────────────────────┘
                         │ COPY --from=builder
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                  Stage 2: Production                         │
│  ┌────────────────────────────────────────────────────┐     │
│  │  FROM node:18-alpine                               │     │
│  │  WORKDIR /app                                      │     │
│  │  COPY --from=builder /app/dist ./dist              │     │
│  │  COPY --from=builder /app/node_modules ./node_modules│  │
│  │  CMD ["node", "dist/server.js"]                    │     │
│  │                                                    │     │
│  │  Size: 150 MB (production only)                    │     │
│  └────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

## 5. Docker Volume Types

```
┌─────────────────────────────────────────────────────────────┐
│                    Volume Types                              │
│                                                              │
│  1. Named Volume                                            │
│  ┌────────────────────────────────────────────────────┐     │
│  │  /var/lib/docker/volumes/myvolume/_data            │     │
│  │                    ▲                                │     │
│  │                    │ mount                          │     │
│  │              ┌─────┴─────┐                         │     │
│  │              │ Container │                         │     │
│  │              │ /app/data │                         │     │
│  │              └───────────┘                         │     │
│  └────────────────────────────────────────────────────┘     │
│                                                              │
│  2. Bind Mount                                              │

│  │  /home/user/myapp                                  │     │
│  │         ▲                                          │     │
│  │         │ bind mount                               │     │
│  │   ┌─────┴─────┐                                    │     │

│  │   └───────────┘                                    │     │
│  └────────────────────────────────────────────────────┘     │
```

## 6. Azure Container Registry (ACR) Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    Developer Workstation                     │
│                                                              │
│  docker build -t myapp:v1 .                                 │
│  docker tag myapp:v1 myacr.azurecr.io/myapp:v1             │
│  docker push myacr.azurecr.io/myapp:v1                     │
└────────────────────────┬────────────────────────────────────┘
                         │ HTTPS
                         ▼
┌─────────────────────────────────────────────────────────────┐
│           Azure Container Registry (ACR)                     │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │  Repositories                                      │     │
│  │  ├── myapp:v1, v2, latest                         │     │
│  │  ├── api:v1, v2                                   │     │
│  │  └── frontend:v1                                  │     │
│  └────────────────────────────────────────────────────┘     │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │  Features                                          │     │
│  │  ├── Geo-replication (Premium)                    │     │
│  │  ├── Image scanning (Defender)                    │     │
│  │  ├── Webhooks                                     │     │
│  │  ├── Tasks (CI/CD)                                │     │
│  │  └── Private Link                                 │     │

└────────────────────────┬────────────────────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         │               │               │

```

```
┌─────────────────────────────────────────────────────────────┐
│                    Azure Subscription                        │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │              Resource Group                        │     │
│  │                                                    │     │
│  │  ┌──────────────────────────────────────────┐     │     │
│  │  │     Container Group (Pod-like)           │     │     │
│  │  │                                          │     │     │
│  │  │  ┌────────────┐  ┌────────────┐         │     │     │
│  │  │  │ Container1 │  │ Container2 │         │     │     │
│  │  │  │   (App)    │  │ (Sidecar)  │         │     │     │
│  │  │  │            │  │  (Logger)  │         │     │     │
│  │  │  └────────────┘  └────────────┘         │     │     │
│  │  │                                          │     │     │
│  │  │  Shared:                                 │     │     │
│  │  │  - Network (IP Address)                  │     │     │
│  │  │  - Storage (Volumes)                     │     │     │
│  │  │  - Lifecycle                             │     │     │
│  │  └──────────────────────────────────────────┘     │     │
│  │                                                    │     │
│  │  ┌──────────────────────────────────────────┐     │     │
│  │  │        Azure Files Volume                │     │     │
│  │  │        (Persistent Storage)              │     │     │
│  │  └──────────────────────────────────────────┘     │     │
│  └────────────────────────────────────────────────────┘     │
│                                                              │
│  Integration:                                               │
│  ├── Virtual Network (VNet injection)                       │
│  ├── Azure Monitor (Logs & Metrics)                         │
│  ├── Azure Key Vault (Secrets)                              │
│  └── Azure Files (Persistent volumes)                       │
└─────────────────────────────────────────────────────────────┘
```

## 8. Container to AKS Migration Path

```
┌─────────────────────────────────────────────────────────────┐
│                  Phase 1: Monolith                           │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │         Single VM / App Service                    │     │
│  │  ┌──────────────────────────────────────────┐     │     │
│  │  │        Monolithic Application            │     │     │
│  │  │  (All features in one codebase)          │     │     │
│  │  └──────────────────────────────────────────┘     │     │
│  └────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────┘
                         │
                         ▼ Containerize
┌─────────────────────────────────────────────────────────────┐
│              Phase 2: Containerized Monolith                 │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │         Azure Container Instances                  │     │
│  │  ┌──────────────────────────────────────────┐     │     │
│  │  │    Docker Container (Monolith)           │     │     │
│  │  │    - Easier deployment                   │     │     │
│  │  │    - Portable                            │     │     │
│  │  └──────────────────────────────────────────┘     │     │
│  └────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────┘
                         │
                         ▼ Break into services
┌─────────────────────────────────────────────────────────────┐
│              Phase 3: Microservices on AKS                   │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │         Azure Kubernetes Service (AKS)             │     │
│  │                                                    │     │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐        │     │
│  │  │ Service1 │  │ Service2 │  │ Service3 │        │     │
│  │  │  (Auth)  │  │  (API)   │  │  (Data)  │        │     │
│  │  └──────────┘  └──────────┘  └──────────┘        │     │
│  │                                                    │     │
│  │  Features:                                         │     │
│  │  - Auto-scaling                                    │     │
│  │  - Load balancing                                  │     │
│  │  - Self-healing                                    │     │
│  │  - Rolling updates                                 │     │
│  └────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────┘
```
