## 2. AKS Networking Models

### Azure CNI (Advanced)
```
┌─────────────────────────────────────────────────────────────────┐
│                    Virtual Network (10.0.0.0/16)                 │
│                                                                  │
│  ┌────────────────────────────────────────────────────────┐     │
│  │         AKS Subnet (10.0.1.0/22)                       │     │
│  │         Available IPs: ~1000                           │     │
│  │                                                        │     │
│  │  Node 1 (10.0.1.4)                                    │     │
│  │  ├── Pod1 (10.0.1.10) ◄─── Direct VNet IP            │     │
│  │  ├── Pod2 (10.0.1.11) ◄─── Direct VNet IP            │     │
│  │  └── Pod3 (10.0.1.12) ◄─── Direct VNet IP            │     │
│  │                                                        │     │
│  │  Node 2 (10.0.1.5)                                    │     │
│  │  ├── Pod4 (10.0.1.20) ◄─── Direct VNet IP            │     │
│  │  └── Pod5 (10.0.1.21) ◄─── Direct VNet IP            │     │
│  │                                                        │     │
│  │  Benefits:                                            │     │
│  │  ✓ Pods accessible from VNet                         │     │
│  │  ✓ No NAT required                                   │     │
│  │  ✓ Network policies supported                        │     │
│  │  ✗ Higher IP consumption                             │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                  │
│  ┌────────────────────────────────────────────────────────┐     │
│  │    Other Subnets can directly access pods             │     │
│  │    - App Gateway: 10.0.5.0/24                         │     │
│  │    - VMs: 10.0.6.0/24                                 │     │
│  └────────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────────┘
```

## 3. AKS Deployment Strategies

### Blue-Green Deployment
```
┌─────────────────────────────────────────────────────────────────┐
│                    AKS Cluster                                   │
│                                                                  │
│  Step 1: Blue (Current) is live                                 │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Service (LoadBalancer)                                │     │
│  │  selector: version=blue                                │     │
│  │              │                                         │     │
│  │              ▼                                         │     │
│  │  ┌──────────────────────┐  ┌──────────────────────┐   │     │
│  │  │  Blue Deployment     │  │  Green Deployment    │   │     │
│  │  │  version=blue        │  │  version=green       │   │     │
│  │  │  replicas: 3         │  │  replicas: 3         │   │     │
│  │  │  ✓ Active (100%)     │  │  ○ Standby (0%)      │   │     │
│  │  └──────────────────────┘  └──────────────────────┘   │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                  │
│  Step 2: Switch traffic to Green                                │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Service (LoadBalancer)                                │     │
│  │  selector: version=green  ◄─── Update selector        │     │
│  │              │                                         │     │
│  │              ▼                                         │     │
│  │  ┌──────────────────────┐  ┌──────────────────────┐   │     │
│  │  │  Blue Deployment     │  │  Green Deployment    │   │     │
│  │  │  version=blue        │  │  version=green       │   │     │
│  │  │  replicas: 3         │  │  replicas: 3         │   │     │
│  │  │  ○ Standby (0%)      │  │  ✓ Active (100%)     │   │     │
│  │  └──────────────────────┘  └──────────────────────┘   │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                  │
│  Step 3: Remove Blue (optional)                                 │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  kubectl delete deployment blue-deployment             │     │
│  └────────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────────┘
```
